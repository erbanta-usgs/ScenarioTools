Model-grid shapefile

 Top  Previous  Next

Both Scenario Manager and Scenario Analyzer make use of a shapefile containing polygons representing all model cells in an areal view of the model grid (that is, all cells in one model layer).

 

The shapefile attribute table needs to include one attribute named “row” or “rownum,” which contains the one-based row index for each model cell. The attribute table also needs contain another attribute named  “col,” “colnum,” “column,” or “columnnum,” which contains the one-based column index for each model cell.

 

Scenario Manager uses a model-grid shapefile to generate model-input files from points, polylines, or polygons representing groundwater-system features or areas of hydrologic significance stored in other shapefiles.

 

Scenario Analyzer uses the model-grid shapefile for making maps. It uses the model-grid shapefile for positioning contours on contour map layers or drawing symbology on color-fill map layers.
